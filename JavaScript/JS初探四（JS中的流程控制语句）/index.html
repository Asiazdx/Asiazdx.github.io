JS初探四（JS中的控制流程语句）

发表于 2016-08-04 | 分类于 JavaScript


其实，这一章很简单，稍微有一点编程基础的同学大能够理解。嗯~，切入正题，简单介绍一下JS中的基础语句。

一、if语句

格式如下：

逻辑：如果if语句中的表达式的值是真就执行里面的语句，如果为假就不执行。
==注意==：可以嵌套，嵌套不要超过6层，一般也就三层。

二、if-else语句

格式：

逻辑：如果表达式的值是真则执行语句1，如果表达式的值是假则执行语句2。

三、if-else-if语句

格式：

逻辑：如果表达式1的值为真则执行语句1，否则执行表达式2，如果表达式2的值为真则执行语句2，否则执行表达式3，如果表达式3的值为真则执行语句3，否则继续……
每个else都对他上面所有表达式的否定。

四、switch语句和break语句

逻辑：执行的expression表达式得到的结果与下面的case的value值进行比较，如果有相同的则执行case后面的语句，之后再执行break。否则继续往下找，直到找到default为止。

五、while语句

格式：

while语句属于前测试循环语句，也就是说，在循环体内的代码被执行之前，就会对出口条件求值。
当表达式expression是真值时则循环执行statement，直到expression的值为假为止；如果是假值，那么程序将跳过循环。

六、do while语句

格式：

do while语句是后测试循环，即退出条件在执行循环内部的代码之后计算。这意味着在计算表达式之前，至少会执行循环主体一次。

七、for语句

for语句提供了一种比while语句更加方便的循环控制结构，用for循环来重复执行一些代码的好处是循环控制结构更加清晰。

大部分的循环都具有特定的计数器变量，计数器的三个关键操作是初始化、检测和更新。for语句将这三步明确声明为循环语法的一部分，各自使用一个表达式来表示。

八、for-in语句

用来枚举对象的属性，如果要迭代的对象是null或者是undefined，那么就会抛出异常。

==注意==:for-in循环并不会遍历对象的所有属性，只有可枚举（enumberable）的属性才会遍历到。

九、break语句和continue语句

break专门用来就跳出switch语句和循环语句的。
break后面的代码是不会被执行的。
break只跳出距离它最近的那一次循环或switch。
用来跳过本次循环，继续下一次循环，通常用在for循环内部。
